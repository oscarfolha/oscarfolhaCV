# Copilot Instructions for Óscar Folha Portfolio

## Project Overview
This is a premium, multilingual portfolio website built with Next.js 16.2.4, TypeScript, and Tailwind CSS v4. It features 6 language support (English, Portuguese, Spanish, French, German, Chinese), dark/light mode themes, smooth animations with Framer Motion, and responsive design.

## Tech Stack
- **Framework**: Next.js 16.2.4 with App Router
- **Language**: TypeScript (strict mode enabled)
- **Styling**: Tailwind CSS v4 with custom design tokens
- **Animations**: Framer Motion v11+
- **Internationalization**: next-intl v4.11.0
- **UI Components**: Custom React components with motion effects
- **Build Tools**: ESLint, Prettier

## Key Conventions

### Coding Standards
1. **Components**: Functional components with TypeScript types
2. **Props**: Always define explicit prop types, avoid `any`
3. **Naming**: 
   - Components use PascalCase (e.g., `HeroSection`)
   - Functions/constants use camelCase
   - CSS classes use lowercase with hyphens
4. **Imports**: Use absolute imports from `@/` (configured in tsconfig)
5. **Error Handling**: Wrap i18n calls to prevent type errors

### File Structure
```
src/
├── app/                    # App Router pages and layout
├── components/
│   ├── ui/                # Reusable UI components (Button, Forms, etc.)
│   └── layout/            # Layout components (Navbar, Footer, etc.)
├── sections/              # Page sections (Hero, About, Experience, etc.)
├── i18n/
│   ├── messages/          # Translation JSON files (6 languages)
│   └── request.ts         # i18n configuration
├── utils/
│   ├── site-config.ts     # Site metadata and social links
│   └── cn.ts              # Tailwind class merging utility
└── styles/                # Global styles and CSS animations
```

### Styling Guidelines
1. **Tailwind CSS**: Use Tailwind utility classes, not custom CSS when possible
2. **Dark/Light Mode**: Use `light:` and `dark:` prefixes for theme variants
   - Dark text: `text-foreground` (light gray)
   - Light text: `text-muted` (medium gray)
   - Accent color: `text-sky-*` classes
3. **Colors**: Reference CSS variables in globals.css for brand consistency
4. **Responsive**: Use `sm:`, `md:`, `lg:` prefixes; design mobile-first

### Component Patterns
- **Sections**: Import translations with `const { t } = await getTranslations()`
- **Reveal Animation**: Use `<Reveal>` wrapper for scroll animations
- **Icon Containers**: Use `bg-sky-300/12` dark mode, `light:bg-sky-200/20` light mode
- **Text Contrast**: 
  - Dark mode: `text-sky-100` for accent text
  - Light mode: `light:text-sky-950` for sufficient contrast

### Internationalization (i18n)
1. **Translation Files**: Located in `src/i18n/messages/`
   - `en.json`, `pt.json`, `es.json`, `fr.json`, `de.json`, `zh.json`
2. **Key Structure**: Hierarchical (e.g., `hero.title`, `nav.about`, `social.email`)
3. **Usage**: `const { t } = await getTranslations()` then `t("section.key")`
4. **Consistency**: Always update all 6 language files in sync

### Animation Principles
1. **Duration**: Use 0.3-0.8s for most transitions
2. **Easing**: Prefer `ease-out` for natural feeling. Motion config:
   ```js
   transition={{ duration: 0.5, ease: "easeOut" }}
   ```
3. **Performance**: Use `will-change` class for GPU acceleration on heavy animations
4. **Accessibility**: Respect `prefers-reduced-motion` in critical animations

### Links Behavior
- **External links** (starting with `http`): Open in new tab (`target="_blank"`)
- **Internal anchor links** (starting with `#`): Navigate within page
- **Email links** (`mailto:`): Open default email client
- **Implementation**: Button component auto-detects external links

## Common Tasks

### Add a New Section
1. Create component in `src/sections/SectionName.tsx`
2. Use `getTranslations()` for translations
3. Wrap animations with `<Reveal>` component
4. Add translations to all 6 JSON files
5. Import and render in `app/[locale]/page.tsx`

### Add Translations
1. Add keys to all 6 JSON files in `src/i18n/messages/`
2. Use consistent key structure: `section.key`
3. Maintain same structure across all languages
4. Test with language switcher

### Update Site Metadata
1. Edit `src/utils/site-config.ts`
2. Update `siteMetadata`, `email`, `socialLinks`
3. Changes automatically reflect across site

### Fix Light Mode Issues
If elements invisible in light mode:
1. Add `light:` variants to dark mode tailwind classes
2. Use `light:text-sky-950` for text on light backgrounds
3. Use `light:bg-sky-200/20` for light-colored backgrounds
4. Test with theme toggle in top-right

## Testing & Validation
1. **Build test**: `npm run build` should complete without errors
2. **Link test**: Click all external links, verify they open in new tabs
3. **i18n test**: Switch between all 6 languages, verify content updates
4. **Theme test**: Toggle dark/light mode, verify all text is readable
5. **Responsive test**: Resize browser, verify layout adapts

## Debugging Tips
1. **TypeScript errors**: Run `npm run build` to see config errors
2. **Styling issues**: Check Tailwind class spelling and theme variants
3. **i18n errors**: Verify key exists in all 6 language files
4. **Build fails**: Clear `.next` folder and rebuild
5. **Images not loading**: Verify paths in `public/gallery/` directory

## Best Practices
1. ✅ Always use TypeScript types, avoid `any`
2. ✅ Maintain consistent code style (ESLint enforced)
3. ✅ Test all 6 languages when adding translations
4. ✅ Optimize images: use JPEG for photos, optimize file size
5. ✅ Write semantic HTML: use proper heading hierarchy
6. ✅ Preserve accessibility: images need alt text, links need descriptive text
7. ✅ Keep component files focused: one main component per file

## Anti-patterns to Avoid
1. ❌ Hardcoding text without translations
2. ❌ Using inline styles instead of Tailwind
3. ❌ Missing light: variants for theme support
4. ❌ Not testing on mobile/responsive sizes
5. ❌ Committing node_modules or .next folder
